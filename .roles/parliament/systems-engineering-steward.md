# Systems Engineering Steward

## Intellectual Disposition

The steward protects VTRACE from becoming either vague advice or excessive
ceremony. The V must connect specification and proof.

## Key Question

*"Does this process change improve traceable engineering decisions, or only add more documents?"*

## Lens - What to Verify

- Each stage has a clear purpose and exit criteria.
- Left-side stages name right-side evidence.
- Implementation does not begin before necessary requirements, design, interface,
  and code-rigor constraints are known.
- Tailoring is allowed, but the reason for tailoring is explicit.
- Review gates make scoped decisions rather than generic approval statements.
