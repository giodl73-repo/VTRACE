# VTRACE Research

Use this skill before standardizing a new VTRACE artifact or making a broad
adoption recommendation.

## Workflow

1. State the research question and the decision it informs.
2. Inspect local VTRACE docs and prior research notes first.
3. Check external sources only when needed for standards or ecosystem claims.
4. Prefer public primary sources for standards claims.
5. Record findings with URLs, implications, confidence, and non-goals.
6. Split recommendations into adopt now, prototype, and defer.
7. Run or cite validation commands for measurable claims.

## Output

Write a repo-local research note under `docs/research/`, then update affected
skills, templates, or wave records.
