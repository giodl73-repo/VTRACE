# Mission

## Scope

Repo: VTRACE

## Mission Need

| ID | Need | Success Criteria | Status |
|---|---|---|---|
| NEED-001 | VTRACE shall give existing software repos a practical V-model adoption path from intent through implementation evidence. | A target repo can create stage artifacts, derive work packages, run gates, and record evidence without inventing process from scratch. | accepted |
| NEED-002 | VTRACE shall remain source-grounded without claiming NASA compliance or endorsement. | Framework claims cite source categories and stay framed as adaptation. | accepted |
| NEED-003 | VTRACE shall be usable by agents and maintainers across heterogeneous repos. | Skills, templates, roles, and examples define actionable steps and stop conditions. | accepted |

## Constraints

- Keep the repo documentation-first until artifact contracts are stable.
- Preserve pointer-first source custody.
- Keep child repo changes separate from TRACKER submodule pointer updates.
- Avoid heavyweight compliance language unless a target repo explicitly needs it.

## Non-Goals

- VTRACE is not an official NASA project.
- VTRACE does not certify compliance.
- VTRACE does not replace domain-specific safety, legal, security, or assurance review.
