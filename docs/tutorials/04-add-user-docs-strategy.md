# Tutorial 04: Add User Docs Strategy

Goal: create a user-facing docs package that is traceable to VTRACE artifacts.

1. Add `docs/vtrace/COMMUNICATIONS_STRATEGY.md`.
2. Add `COMMS-*` rows for `docs/README.md`, concepts, how-to guides,
   tutorials, examples, traces, decks, and `docs/CORPUS.md` as needed.
3. For each row, name source IDs and the user question the docs answer.
4. Create only the docs surfaces the repo actually needs.
5. Run `vtrace comms plan <repo>`.
6. Run `vtrace validate <repo>`.

Expected result: user docs explain accepted behavior, while requirements and
specs remain the controlled source of engineering truth.
