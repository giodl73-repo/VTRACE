# VTRACE Docs

This directory contains VTRACE user-facing documentation and framework
reference material.

Use `docs/vtrace/` when you need the controlled self-trace proof package. Use
the user docs below when you need to understand, adopt, or teach VTRACE.

## Documentation Map

| Surface | Audience | Purpose |
|---|---|---|
| `concepts/` | adopters, maintainers, reviewers | Explain the mental model behind VTRACE. |
| `how-to/` | repo maintainers, agents | Provide focused task instructions. |
| `tutorials/` | new adopters | Teach VTRACE through a staged learning path. |
| `examples/` | implementers, reviewers | Show copyable artifact patterns and expected output. |
| `traces/` | reviewers, portfolio owners | Walk through end-to-end proof chains. |
| `decks/` | stakeholders, adoption leads | Provide presentation outlines for rollout and review. |
| `framework/` | process owners | Define reusable VTRACE framework guidance. |
| `research/` | source reviewers | Record source-grounded research notes. |
| `vtrace/` | VTRACE maintainers | Prove VTRACE with VTRACE-controlled artifacts. |
| `CORPUS.md` | docs owners | Govern ownership, cadence, and update obligations. |

## Recommended Start

1. Read `concepts/vtrace-in-one-page.md`.
2. Follow `tutorials/01-minimum-slice.md`.
3. Use `how-to/adopt-vtrace-in-an-existing-repo.md`.
4. Follow `how-to/run-a-delivery-dashboard.md` when a repo needs a compact operating view over active VTRACE work.
5. Run `vtrace comms plan <repo>` when a repo needs user-facing docs.
6. Check `traces/communications-strategy-trace.md` to see how this docs package
   traces back to VTRACE requirements and evidence.
