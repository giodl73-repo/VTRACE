# Work Packages And Evidence

A VTRACE work package is the procedural unit of implementation. It turns
accepted intent into a bounded slice that an engineer or agent can execute.

Each work package should name:

- parent IDs,
- affected surfaces,
- entry criteria,
- exit criteria,
- L0/L1/L2 validation expectations,
- required review lanes,
- evidence targets.

Closeout is not just "tests passed." A closed work package should update the
trace matrix, verification rows, validation rows, evidence ledger, review
status, and any user-facing docs affected by the change.
