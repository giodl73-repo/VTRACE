# Adoption Docs Package Deck

## Slide 1: Problem

Repos often have specs, tests, and README fragments, but users still do not
know what to read, what to try, or which examples prove the feature.

## Slide 2: VTRACE Rule

Specs control engineering intent. User docs explain accepted behavior.

## Slide 3: Source Mapping

Mission defines the docs map. CONOPS defines tutorials and walkthroughs.
Requirements and specs define concepts, how-to guides, references, and
examples. Work packages and evidence define traces and release narratives.

## Slide 4: Package

The recommended package is `docs/README.md`, `docs/concepts/`, `docs/how-to/`,
`docs/tutorials/`, `docs/examples/`, `docs/traces/`, optional `docs/decks/`,
and `docs/CORPUS.md`.

## Slide 5: Governance

`COMMUNICATIONS_STRATEGY.md` owns the source-to-docs mapping. `docs/CORPUS.md`
owns surface inventory and update obligations.
