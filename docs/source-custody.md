# Source Custody and Encoding Policy

VTRACE can download and encode standards only when the source rights and
distribution posture allow it. The default pattern is pointer-first: keep source
metadata, URLs, versions, access dates, rights posture, and derived concepts in
this repo; vendor full text only when redistribution is clearly allowed.

## Source Classes

| Class | Examples | Repo Treatment |
|---|---|---|
| Public NASA handbook or directive page | NASA Systems Engineering Handbook, NODIS NPR pages | Store URLs, metadata, source IDs, derived summaries, and encoded concepts. Download only into ignored cache paths unless redistribution is explicitly clear. |
| Public NASA PDF or standard | NASA/SP-2016-6105 Rev2, NASA-STD-8739.8B | Store URL, version, document date, public access posture, and optional checksum for operator-downloaded cache bytes. |
| Copyrighted industry standard | ISO/IEC/IEEE 15288, ISO/IEC/IEEE 29148, IEEE 1012 | Store bibliographic metadata and mapping notes only. Do not vendor text or encode clause-level paraphrases that substitute for the standard. |
| Portfolio-derived method | VTRACE templates and skills | Commit directly under `docs/`, `templates/`, `schemas/`, and `skills/`. |

## Download Rule

Use a three-level policy for every source:

- `pointer_only`: keep metadata and links; no download expected.
- `cache_allowed`: operator may download into ignored local cache for research,
  checksum, and derived concept extraction.
- `redistribution_allowed`: source bytes or excerpts may be committed if the
  source license and repo policy allow it.

VTRACE foundation defaults to `pointer_only` or `cache_allowed`. Committed
source bytes are out of scope until a source-specific rights review is recorded.

## Encoding Rule

Encoding should capture the method VTRACE needs, not reproduce the source:

- source identity and version,
- process or artifact concept,
- software-codebase adaptation,
- expected trace links,
- objective evidence expectations,
- validation or verification method,
- rights posture and citation.

Do not encode large verbatim sections. Do not convert restricted standards into
a free substitute. When a standard is copyrighted, encode only high-level
metadata and locally authored adoption guidance.

## Recommended Cache Layout

Downloaded bytes should live outside committed source history:

```text
target/source-cache/<source-id>/
  source.pdf
  source.html
  checksum.txt
  fetch-metadata.json
```

`target/` is ignored by convention and can be regenerated by an operator.

## First Encoded Source Set

Start with NASA sources because they are public, directly relevant, and enough
to define VTRACE's first framework:

- NASA Systems Engineering Handbook.
- NASA Systems Engineering Handbook appendix material.
- NPR 7123.1D, NASA Systems Engineering Processes and Requirements.
- NASA Software Engineering Handbook.
- NASA-STD-8739.8B, Software Assurance and Software Safety Standard.

Industry standards should remain citation-only until a separate rights and
access decision exists.
