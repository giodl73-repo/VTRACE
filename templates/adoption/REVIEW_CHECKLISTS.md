# Review Checklists

## Scope

Repo or feature:

## Checklist

| Gate | Item | Required | Decision | Evidence / Notes |
|---|---|---|---|---|
| Specification Review | Mission, requirements, specs, unknowns, and validation path are clear. | yes | pending |  |
| Design Review | Architecture, package boundaries, interfaces, and code-rigor constraints are acceptable. | yes / no | pending |  |
| Implementation Readiness Review | Work packages have parent IDs, entry/exit criteria, profiles, and L0/L1/L2 checks. | yes / no | pending |  |
| Work Package Close Review | Implementation surfaces, trace rows, evidence, validation impact, and role lanes are closed. | yes / no | pending |  |
| Test Readiness Review | Procedures, fixtures, expected results, and environment assumptions are ready. | yes / no | pending |  |
| Release/Transition Readiness Review | Evidence supports the readiness claim and open risks are accepted or blocked. | yes / no | pending |  |
