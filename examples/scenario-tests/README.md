# Scenario Tests

Scenario tests stress the VTRACE process without requiring a full target repo.
Each card names the expected stage emphasis, code-rigor need, and review gate.

Use these before changing the process or skills:

1. Read the scenario card.
2. Identify which templates are required.
3. Identify expected trace IDs and review gate outcome.
4. Check whether `vtrace-assess`, `vtrace-adopt`, and `vtrace-gate` instructions
   would lead to the expected result.

These are process tests, not executable tests.
